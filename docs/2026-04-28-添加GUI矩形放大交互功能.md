# 2026-04-28 添加 GUI 矩形放大交互功能

## 1. 目标

为软件中的各个实时图增加统一的矩形放大交互能力，同时满足以下约束：

- 普通实时曲线图与 Time-Space 图都支持左键框选放大
- 放大后实时数据继续刷新，但当前视图范围不被下一帧自动抢回
- 右键提供 `View All` 恢复全视图
- 不引入额外的高频重算或明显卡顿

本次覆盖范围如下：

- `Tab1` 图1：时域曲线图
- `Tab1` 图2：频谱图
- `Tab1` 图3：Monitor 图
- `Tab2` Time-Space 图

## 2. 实现方式

### 2.1 新增统一交互层

新增 [`src/plot_interaction.py`](/D:/OneDrive%20-%20EVER/00_KY/DAS-%E5%99%A8%E4%BB%B6%E6%B5%8B%E8%AF%95%E3%80%81%E4%B8%8A%E4%BD%8D%E6%9C%BA%E5%BC%80%E5%8F%91/BX/PCIe-7821/pcie7821_gui/src/plot_interaction.py)，封装 `ZoomablePlotViewBox`，统一提供：

- 左键拖拽：矩形框选放大
- `Shift + 左键拖拽`：仅沿 X 方向平移
- 滚轮：缩放
- 右键菜单：`View All`

### 2.2 主窗口 3 个曲线图统一接入

在 [`src/main_window.py`](/D:/OneDrive%20-%20EVER/00_KY/DAS-%E5%99%A8%E4%BB%B6%E6%B5%8B%E8%AF%95%E3%80%81%E4%B8%8A%E4%BD%8D%E6%9C%BA%E5%BC%80%E5%8F%91/BX/PCIe-7821/pcie7821_gui/src/main_window.py) 中新增：

- `_create_interactive_plot_widget(plot_key)`
- `_on_plot_manual_range_change(plot_key)`
- `_restore_plot_auto_range(plot_key)`
- `_plot_zoom_locked`

每个 `PlotWidget` 都在创建时绑定自定义 `ViewBox`，并在用户手动交互后记录该图已进入“缩放锁定”状态。

### 2.3 实时刷新与视图范围解耦

关键点不是停止 `setData(...)`，而是停止自动改写 `ViewBox` 的范围。

当前行为为：

1. 用户手动缩放或平移
2. 对应图的 `ViewBox.disableAutoRange()`
3. 后续实时 `setData(...)` 继续执行
4. 视图保持用户放大后的范围

只有在右键点击 `View All` 后，才会重新：

- `enableAutoRange(x=True, y=True)`
- `autoRange(padding=0.0)`

这样可以保证“数据继续更新”和“当前放大结果不丢失”同时成立。

### 2.4 Time-Space 图同步接入

`Time-Space` 组件在 [`src/time_space_plot.py`](/D:/OneDrive%20-%20EVER/00_KY/DAS-%E5%99%A8%E4%BB%B6%E6%B5%8B%E8%AF%95%E3%80%81%E4%B8%8A%E4%BD%8D%E6%9C%BA%E5%BC%80%E5%8F%91/BX/PCIe-7821/pcie7821_gui/src/time_space_plot.py) 内部直接使用同一套 `ZoomablePlotViewBox`，并维护独立的 `_zoom_locked` 状态。

这保证了 `Tab1` 与 `Tab2` 的鼠标交互行为一致，不会出现一边能框选、另一边只能默认拖动的问题。

## 3. 运行时行为

### 3.1 缩放后的刷新行为

- 图像或曲线内容继续更新
- 自动范围刷新被锁定
- 用户可持续观察放大区域内的新数据变化

### 3.2 恢复全图行为

右键 `View All` 后：

- 解除该图的缩放锁定
- 恢复自动范围
- 下一帧开始重新按实时数据全范围显示

### 3.3 频谱图的特殊处理

频谱图原有逻辑在每次刷新时会主动调用 `setXRange(...)` 或重新启用自动范围，这会覆盖用户的手动放大结果。

本次已在频谱刷新逻辑中加入缩放锁定判断，只有在未锁定时才允许自动改写范围。

## 4. 风险与说明

### 4.1 Raw 图仍为实时大曲线

Raw 曲线已经改为保留完整显示数据并依赖 PyQtGraph 的视图裁剪和自动抽样，因此放大时可以保留更多局部细节。

### 4.2 `View All` 是恢复入口

当前没有增加双击恢复等额外交互，统一使用右键 `View All` 作为恢复入口，逻辑更稳定，也便于与 PyQtGraph 默认习惯保持一致。

## 5. 总结

本次修改完成了软件内主要实时图的统一矩形放大交互接入，并重点解决了“放大后被下一帧自动抢回视图”的问题。对于需要持续盯某一小段曲线或某一局部时空区域的场景，交互稳定性已经明显提升。
