# 2026-04-28 本地参数自动保存与恢复

## 1. 目标

为 GUI 增加本地参数自动保存与恢复能力，使软件在下次打开时，能够回到上一次关闭或启动时使用的参数状态。

本次目标包括：

- 启动时自动尝试读取上次参数
- 启动采集前自动保存当前参数
- 关闭窗口时再次保存当前参数
- 持久化文件位置在源码运行和打包运行下都稳定
- 本地参数文件损坏时不影响程序启动

## 2. 实现方式

### 2.1 新增本地参数文件路径解析

在 [`src/main_window.py`](/D:/OneDrive%20-%20EVER/00_KY/DAS-%E5%99%A8%E4%BB%B6%E6%B5%8B%E8%AF%95%E3%80%81%E4%B8%8A%E4%BD%8D%E6%9C%BA%E5%BC%80%E5%8F%91/BX/PCIe-7821/pcie7821_gui/src/main_window.py) 中新增：

- `_get_settings_path()`

路径规则如下：

- 源码运行：项目根目录下 `last_params.json`
- 冻结运行：`exe` 同级目录下 `last_params.json`

### 2.2 新增保存与加载入口

新增以下方法：

- `_save_local_params()`
- `_load_local_params()`
- `_apply_params_to_ui(params)`
- `_merge_dict_into_dataclass(target, values)`

保存格式为：

```json
{
  "version": 1,
  "params": {
    "...": "..."
  }
}
```

### 2.3 持久化内容范围

当前自动保存覆盖：

- `basic`
- `upload`
- `phase_demod`
- `display`
- `time_space`
- `save`

其中本次特别补齐了显示开关类字段：

- `display.waveform_plot_enabled`
- `display.monitor_plot_enabled`

### 2.4 自动触发时机

当前保存时机为：

- 点击 `START` 前
- 窗口关闭时

恢复时机为：

- 主窗口初始化完成后

这样既能覆盖正常采集启动场景，也能覆盖用户只改参数不启动、直接退出程序的场景。

## 3. 运行时行为

### 3.1 首次运行

如果本地不存在 `last_params.json`：

- 使用默认参数启动
- 不报错
- 等到后续保存时自动创建该文件

### 3.2 后续运行

如果存在有效的 `last_params.json`：

- 启动时自动恢复上次参数
- 包括 Time-Space 控件和显示开关
- 恢复后再执行常规的点数、文件大小和控件状态刷新

### 3.3 本地文件异常

如果本地参数文件：

- JSON 格式损坏
- 缺少字段
- 含旧版本字段

当前处理方式为：

- 记录 warning
- 忽略本次恢复
- 保持默认参数继续启动

## 4. 仓库与部署说明

### 4.1 `last_params.json` 属于本地运行态文件

本次已将 `last_params.json` 加入 [`.gitignore`](/D:/OneDrive%20-%20EVER/00_KY/DAS-%E5%99%A8%E4%BB%B6%E6%B5%8B%E8%AF%95%E3%80%81%E4%B8%8A%E4%BD%8D%E6%9C%BA%E5%BC%80%E5%8F%91/BX/PCIe-7821/pcie7821_gui/.gitignore)，避免本地缓存参数混入版本控制。

### 4.2 这不是严格配置版本管理

当前机制定位是“本机最近一次使用参数缓存”，不是团队共享配置，也不负责参数历史版本迁移。对于后续更复杂的配置演进，必要时可再引入显式 schema version 迁移逻辑。

## 5. 总结

本次已完成软件本地参数的自动保存与恢复，持久化路径稳定、恢复链路完整、异常文件可安全退化。对频繁重复启动的软件调试和现场使用场景，能显著减少反复重新设置参数的时间成本。
